# L×Box v1.3.0

Android VPN client powered by [sing-box](https://sing-box.sagernet.org/).

Major release: parser v2 rewritten from scratch, subscription auto-update, AntiDPI mixed-case SNI, haptic feedback.

## ✨ Highlights

- **Parser v2** — full rewrite into sealed `NodeSpec`, 3-layer pipeline, 9 protocols including **new TUIC v5**.
- **Subscription auto-update** — 4 triggers (app start, VPN connected +2min, every hour, VPN stopped) with hard gates against spam.
- **Mixed-case SNI** — new AntiDPI trick to bypass naive exact-match DPI.
- **Haptic feedback** — vibro response on VPN events (default on).
- **Restart warning** is now sticky — doesn't disappear when you cancel the Stop dialog.

---

## 🔧 Parser v2 (spec 026, 5 phases)

Full rewrite of the internal pipeline:

- **Typed sealed `NodeSpec` hierarchy** — 9 protocols: VLESS, VMess, Trojan, Shadowsocks, Hysteria2, **TUIC v5 (new)**, SSH, SOCKS, WireGuard.
- **Polymorphic `emit(vars)`** — WireGuard emits to Endpoint, others to Outbound, with no runtime type checks. Compiler enforces exhaustive switch.
- **Round-trip invariant** — `parseUri(spec.toUri()) ≈ spec` tested for every variant.
- **XHTTP fallback** via sealed `TransportSpec` — compiler won't let you forget the fallback.
- **`ServerList` (sealed)** — `SubscriptionServers` (URL fetch) vs `UserServer` (paste/file/qr/manual). One-shot migration `proxy_sources` → `server_lists` on first read.
- **Functional pipeline** — `parseFromSource(source) → buildConfig(lists, settings) → BuildResult(config, validation, warnings)`.
- **`ValidationResult`** with typed `ValidationIssue`: dangling outbound refs, empty urltest, invalid selector default.
- **~2700 LOC of v1 removed** — `node_parser`, `config_builder`, `source_loader`, `subscription_decoder/fetcher`, `xray_json_parser`, `parsed_node`, `proxy_source`.
- **116 tests** covering models, parser, round-trip, builder, validator, migration, subscription pipeline, e2e.

## 🔄 Subscription auto-update (spec 027)

- **4 triggers:** app start · 2 min after VPN connected · once per hour · immediately on VPN disconnected. Manual refresh (⟳) is the fifth, force-mode.
- **Hard gates against spam:** `minRetryInterval=15min` (persisted via `lastUpdateAttempt`), `maxFailsPerSession=5` (in-memory, thaws on app restart), `perSubscriptionDelay=10s ± 2s` between subscriptions, `_running`/`_inFlight` dedup flags, `inProgress` guard against double-clicks.
- **Crash-safe init sweep** — a stuck `inProgress` on disk (kill -9) is reset to `failed`, fetch allowed after 15-min cooldown.
- **UI in subscription rows** — `124 nodes · 🔄 24h · 🕐 3h ago · (2 fails)` with interval / ago / failcount.
- **Subscription block in detail screen** — URL (tap=copy), Update interval picker (1/3/6/12/24/48/72/168h), Status row, Refresh now button.
- **Rebuild config does NOT trigger HTTP** — local-only assembly from already-loaded nodes.

## 🛡️ AntiDPI: Mixed-case SNI (spec 028)

New toggle in **Settings → DPI Bypass**. Randomises letter case in `server_name` (`WwW.gOoGle.CoM`). Per RFC 6066 SNI is case-insensitive — the server must accept any case; the trick breaks naive exact-match DPI used by regional providers.

- First-hop only, per-outbound independent randomisation, punycode labels (`xn--…`) are left alone.
- **Default off.** Honest help text: `Bypasses simple exact-match DPI; ineffective against GFW-class filtering`.

## 📳 Haptic feedback (spec 029)

Toggle in **App Settings → Feedback**, default **on**. Respects the Android system Touch feedback setting.

| Event | Intensity |
|---|---|
| Tap Start/Stop | Light tick |
| VPN connected | Medium impact |
| User disconnect | Light impact |
| Revoked / heartbeat fail (**first only**) | Heavy impact |
| Manual subscription fetch success/fail | Light / medium |

Auto triggers (periodic update, ping, scroll) don't vibrate. 100 ms throttle prevents spam.

## 🔔 Restart warning — sticky flag

The pink **“Config changed — restart VPN to apply”** badge under Stop is now shown reliably after any scenario (Routing Apply, settings change, Debug import, manual rebuild) and **doesn't disappear when you cancel the Stop dialog**.

Implemented via a derived getter `_needsRestart` on top of the sticky `state.configStaleSinceStart` flag in `HomeState` — cleared **only** on a real tunnel up↔down transition.

## 📝 Misc

- **Subscription title fallback** — if no `profile-title` header, the name is taken from `Content-Disposition: filename=...` (RFC 5987 support, strips `.txt`/`.yaml`/`.json`/`.conf`).
- **User-Agent** for subscription HTTP → `LxBox Android subscription client` (was `SubscriptionParserClient`).
- **DNS rules** — inline `domain_suffix: [ru, xn--p1ai, su]` for Yandex DoH, no intermediate ruleset reference.
- **Local build marker** — `scripts/build-local-apk.sh` builds APK with git describe; About screen shows a pink badge «🧪 LOCAL BUILD · N commits since vX.Y.Z». CI builds are clean.

Grab the APK below and install on Android (8.0+). Allow VPN permission on first launch.
