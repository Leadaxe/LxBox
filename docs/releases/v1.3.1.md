# L×Box v1.3.1

Android VPN client powered by [sing-box](https://sing-box.sagernet.org/).

Patch release: UX polish + critical fixes on top of v1.3.0.

## 🐛 Critical fixes

- **`UserServer` was stuck in an infinite spinner after app restart** — `toJson` only stored `rawBody`, `fromJson` never parsed it back, so `NodeSettingsScreen._load()` saw an empty `nodes` list → `_originalTag` wasn't set → spinner. Now `fromJson` reconstructs nodes via `parseAll(decode(rawBody))`. `rawBody` is the source of truth, nodes are derivable.
- **Detour dropdown in Node Settings didn't persist** — wrote `detour` into the node JSON, but `parseSingboxEntry` doesn't restore that field → save → reparse silently dropped detour. Now persists via `entry.detourPolicy.overrideDetour` (which the builder already applies), saved immediately on dropdown change — no separate Save needed.
- **XHTTP warning was hidden behind TLS-insecure** — `node.warnings.first` was used unconditionally, and `InsecureTlsWarning` (parse-time) overshadowed `UnsupportedTransportWarning('xhttp')` (emit-time). The new `_NodeWarningRow` sorts by severity (error → warning → info).
- **TLS-insecure severity downgraded to info (grey)** — providers often set the flag intentionally (REALITY, IP literals, self-signed) and shouldn't crowd out genuine warnings. The warning banner at the top of the detail screen now counts only actionable severities.

## ✨ UI polish

### NodeRow new layout

```
┌────────────────────────────────────────────────────┐
│ Tag                                                │
│ [ACTIVE✓]  PROTOCOL                          50MS  │
└────────────────────────────────────────────────────┘
                                                ^right-aligned
```

- **ACTIVE** — green pill (replaces the text "ACTIVE · 50MS").
- **Protocol** — grey: `VLESS`, `Hy2`, `WG`, `TUIC`, `SS`. For `auto-proxy-out` (urltest) it shows the proto of the **selected** node: `→ BL: Frankfurt   VLESS`. TLS suffix dropped — TLS is default for most protocols, labelling each = noise.
- **Ping** — right-aligned, colour by latency (`<200ms` green, `<500` orange, `>500`/err red).

### Long-press → Copy URI

New item in the popup menu — the original `vless://` / `wireguard://` / etc via `node.toUri()` (round-trip parser v2). `Copy server` renamed to `Copy server (JSON)` for clarity. Control-only nodes show a snackbar `No source URI for this node`.

### NodeSettingsScreen

- **Editable Tag field** — separate TextField under Server (previously the tag was buried in the JSON editor). AppBar title updates live.
- **Mark as detour server** switch — toggles the `⚙ ` prefix on the tag. Stored directly in `tag` (no separate flags in JSON), visually highlights detour servers in the node list and in the Override-detour picker.

### Smart add flow

- **Empty input + tap `+` = paste from clipboard** — if the input field is empty, `+` opens the paste flow (type analysis + confirmation dialog). One tap instead of two.
- **Auto-regenerate config after `addFromInput`** — after paste/QR/file, `generateConfig` + `saveParsedConfig` + snackbar runs automatically. No more manual ⟳.

### Subscription row subtitle

- **Unified `<PROTOCOL> server` for UserServer** (e.g. `WIREGUARD server`, `VLESS server`). Previously different text depending on import form (`WireGuard config` / `Direct link` / `JSON outbound`) — described the form of paste, not the substance. After an app restart (when `entry.status` was lost) it showed a meaningless "1 node" for single-node entries.

## ♻️ Rename: `UserServers` → `UserServer`

Historically plural, but always exactly 1 node (paste/QR/file/manual). Renamed to singular for clarity. 10 files touched (1 model, 2 controllers, 4 screens, 4 tests, 1 migration). **JSON discriminator `'type': 'user'` preserved — no migration needed.**
