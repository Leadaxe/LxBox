### Самые полезные видимые изменения (в порядке ценности)

1. **Exp-backoff retry на fetch подписок и rule-set'ов.** Transient-сбои сети (DPI-сброс, DNS-fail, DDoS-guard) больше не убивают апдейт с первой попытки. 3 попытки с 1s/3s паузами; 4xx = permanent, не ретраим.
2. **Top-level error boundary.** `FlutterError.onError` / `PlatformDispatcher.onError` роутят uncaught в AppLog. Красный экран заменён на компактный fallback с "Check Debug → Logs".
3. **Human-readable ошибки.** Вместо `Exception: HTTP 503 for https://...` — `Server error (503) — provider is down, try later`. Подхвачено в SubscriptionController и SubscriptionDetailScreen.
4. **URL masking audit.** Найдено и закрыто **4 leak-сайта**: re-hydrate warning, inProgress-skip debug, 60-char truncation shortUrl, addFromInput input preview. Токены провайдера больше не попадают в логи.
5. **Pull-to-refresh на Subscriptions + Onboarding card.** Пустой экран теперь показывает 3-step «Getting started» + public-test-servers. На заполненном — тяни вниз для обновления.
6. **Parse hints.** Если подписка пришла успешно, но 0 нод — диагностируем: HTML-страница / Clash YAML / полный sing-box config / plain-text ошибка провайдера. Юзер сразу видит «почему».
7. **Share subscription URL (masked by default).** Из context-menu подписки — диалог с выбором `masked` / `full` с предупреждением.
8. **Debug logs: full-text search** + уже существовавшие source/level фильтры.
9. **Reset fail-count action.** Если провайдер вернулся в строй после фриза (5 фейлов), юзер может разморозить одним тапом без app-restart.
10. **«2h ago»/«yesterday»** + единый `relative_time.dart` helper.
