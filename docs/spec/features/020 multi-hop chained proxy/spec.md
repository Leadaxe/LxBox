# 020 — Multi-hop / Chained Proxy UI

## Контекст

Сейчас chained proxy (jump server) поддерживается только на уровне парсера (012):
подписка может содержать ноду с `dialerProxy` / `detour`, и ConfigBuilder эмитит
jump-outbound автоматически. Но пользователь:

- Не видит, что нода использует цепочку
- Не может собрать цепочку вручную (A → B → C)
- Не может изменить jump-сервер у существующей ноды
- Не может построить multi-hop (больше одного хопа)

## Концепция

**Chain** — упорядоченный список нод (hop'ов), через которые трафик проходит последовательно.
Каждый hop — это outbound из существующего списка нод. Последний hop — exit node.

```
Client → Hop1 (entry) → Hop2 → ... → HopN (exit) → Internet
```

sing-box реализует это через поле `detour` у каждого outbound'а:
```
HopN.detour = Hop(N-1).tag
Hop(N-1).detour = Hop(N-2).tag
...
Hop1 — без detour (прямое подключение)
```

### Хранение

В `boxvpn_settings.json`:

```json
"chains": [
  {
    "tag": "US via RU",
    "hops": ["ru-server-1", "us-server-3"]
  },
  {
    "tag": "Triple hop",
    "hops": ["de-entry", "nl-relay", "us-exit"]
  }
]
```

- `tag` — имя цепочки (отображается как нода в группе)
- `hops` — массив тегов нод **в порядке прохождения** (первый = entry, последний = exit)
- Минимум 2 хопа

### Применение в ConfigBuilder

После генерации всех outbound'ов:

1. Для каждого chain создать **копии** outbound'ов хопов с уникальными тегами:
   - `chain-tag/hop-tag` (например `US via RU/ru-server-1`)
2. Проставить `detour` по цепочке:
   - `hops[1].detour = hops[0].tag`
   - `hops[2].detour = hops[1].tag`
   - и т.д.
3. Добавить **последний** hop (exit) в proxy-группу с тегом цепочки
4. Промежуточные хопы — добавить как outbound'ы, но **не** в группу

### UI

#### Главный экран — индикация цепочки

- Ноды-цепочки отображаются в общем списке с иконкой `🔗` или `link` icon
- Subtitle показывает хопы: `ru-server-1 → us-server-3`
- Ping работает как обычно (проверяет весь путь через exit node)

#### Chain Editor (новый экран)

Доступ: long press на цепочке → Edit chain, или "+" → New chain

**Экран:**
```
┌─────────────────────────────┐
│ Chain name: [US via RU    ] │
│                             │
│  ① ru-server-1         [×] │
│  ② us-server-3         [×] │
│                             │
│  [+ Add hop]                │
│                             │
│  [Save]          [Delete]   │
└─────────────────────────────┘
```

- Список хопов — ReorderableListView (drag для изменения порядка)
- [×] удаляет хоп из цепочки
- [+ Add hop] — открывает picker со всеми доступными нодами (кроме уже добавленных в эту цепочку и других цепочек)
- Валидация: минимум 2 хопа, уникальные теги
- Save → сохраняет в settings, пересобирает конфиг

#### Контекстное меню ноды (long press)

Добавить пункт для обычных нод:
- **Add to chain** → выбор существующей цепочки или "New chain"

Для нод-цепочек:
- **Edit chain** → Chain Editor
- **Delete chain** → удаляет цепочку (ноды остаются)

#### Создание цепочки

Два пути:
1. "+" на главном экране / drawer → "New chain" → Chain Editor
2. Long press на ноде → "Add to chain" → "New chain" → автоматически добавляет ноду как первый хоп

### Ограничения и валидация

- Нода не может быть одновременно в двух цепочках (избежать циклов)
- Нода-цепочка не может быть хопом в другой цепочке
- Все хопы должны существовать на момент генерации конфига — если нода удалена из подписки, показать warning и предложить удалить хоп
- Максимум 5 хопов (больше — бессмысленно и создаёт latency)

### Совместимость с существующими jump-нодами

Ноды, пришедшие из подписки с `dialerProxy` (012), отображаются как цепочки автоматически:
- ConfigBuilder определяет ноду с `jump != null` → создаёт implicit chain
- Пользователь может "Edit chain" → разорвать или модифицировать цепочку
- После ручного редактирования — сохраняется в `chains[]`, оригинальный `jump` игнорируется

## Критерии приёмки

- [ ] Пользователь может создать цепочку из 2+ нод через UI.
- [ ] Цепочка отображается в списке нод с иконкой и хопами в subtitle.
- [ ] Порядок хопов меняется drag & drop.
- [ ] ConfigBuilder генерирует корректные outbound'ы с `detour` по цепочке.
- [ ] Ping цепочки работает (весь путь).
- [ ] Ноды с `jump` из подписки отображаются как цепочки.
- [ ] Удаление цепочки не удаляет сами ноды.
- [ ] Валидация: нет циклов, нет дублей, минимум 2 хопа.
- [ ] VPN подключается и работает через multi-hop цепочку.
