# 022 — Node Filter for Auto-Proxy Group

## Контекст

Сейчас все ноды из всех подписок попадают в auto-proxy-out (urltest) и другие группы. Пользователь не может исключить плохие/ненужные ноды — только удалить подписку целиком.

Нужна возможность выбирать какие ноды включены в группу, не трогая подписки.

## Концепция

Экран с полным списком нод и чекбоксами. По умолчанию все включены. Пользователь снимает галочки с ненужных. Исключённые ноды не попадают в конфиг при генерации.

### Хранение

В `SharedPreferences`:

```json
"excluded_nodes": ["tag1", "tag2", "tag3"]
```

Хранятся только **исключённые** теги (инвертированная логика — по умолчанию всё включено, не нужно хранить огромный список включённых).

### Применение в ConfigBuilder

В `_buildPresetOutbounds`:
- Загрузить `excluded_nodes` из настроек
- Отфильтровать `allNodes` — убрать ноды с тегами из excluded
- Не генерировать outbound для исключённых нод

**Функциональность:**
- Чекбокс на каждой ноде (включена/исключена)
- Поиск по имени ноды
- Кнопки: Select All / Deselect All
- Счётчик включённых/всего
- Группировка по подписке (опционально, v2)
- При Apply — сохраняет excluded list, пересобирает конфиг
- Последний пинг рядом с нодой (если есть данные)

### Поведение при обновлении подписки

- Новые ноды (которых не было раньше) — включены по умолчанию (их нет в excluded)
- Удалённые ноды — автоматически пропадут, их теги в excluded не мешают
- Переименованные ноды — потеряют excluded статус (по тегу), это ок

## Критерии приёмки

- [ ] Пользователь может открыть список всех нод с чекбоксами
- [ ] По умолчанию все ноды включены
- [ ] Снятие галочки исключает ноду из конфига
- [ ] Поиск по имени ноды работает
- [ ] Select All / Deselect All работает
- [ ] При Apply конфиг пересобирается без исключённых нод
- [ ] Новые ноды после обновления подписки включены по умолчанию
- [ ] Счётчик показывает N / Total included
