## 1. Цель

Мобильный UI для управления подписками (добавление, удаление, обновление) и настройками (переменные из wizard template), с автоматической перегенерацией конфига.

## 2. Экран подписок (Subscriptions)

### 2.1 Список подписок

- Каждая подписка отображается как карточка:
  - URL (или «Direct link» для прямых ссылок)
  - Количество распарсенных нод
  - Статус: загружена / ошибка / не обновлялась
  - Кнопка удаления

### 2.2 Добавление подписки

Способы добавления:
1. **URL подписки** — текстовое поле + кнопка «Add»
2. **Вставка из буфера** — кнопка paste, автоопределение: URL подписки или direct link
3. **Direct link** — одиночная ссылка `vless://`, `vmess://` и т.д.

После добавления:
- Если URL — fetch → decode → parse → показать количество нод
- Если direct link — parse → показать имя узла
- При ошибке — сообщение (bad URL, parse error, network error)

### 2.3 Действия

- **Обновить все** — re-fetch всех подписок, перегенерация конфига
- **Удалить** — свайп или кнопка, подтверждение, перегенерация конфига
- **Генерировать конфиг** — явная кнопка для пересборки конфига из текущих подписок + настроек

### 2.4 Состояние при обновлении

- Прогресс: `Downloading 1/3...`, `Parsing 2/3...`
- Блокировка UI действий во время обновления
- Результат: `Updated: 150 nodes from 3 subscriptions`

## 3. Экран настроек (Settings)

### 3.1 Источник переменных

Из `wizard_template.json → vars[]` с фильтрацией:
- Только `wizard_ui: "edit"`
- Только для текущей платформы (Android: без `platforms: ["darwin"]` и т.д.)

### 3.2 Виджеты по типу переменной

| Тип | Виджет |
|-----|--------|
| `bool` | `SwitchListTile` |
| `enum` | `DropdownButton` |
| `text` | `TextField` |
| `secret` | `TextField` + кнопка генерации случайного значения |

### 3.3 Группировка

Переменные разделены `separator: true` записями в массиве vars. В UI — `Divider` между группами.

### 3.4 Selectable Rules

Отдельная секция на экране настроек:
- Список предустановленных правил маршрутизации
- Каждое правило: `SwitchListTile` (label + description)
- Значения по умолчанию из `default: true` в шаблоне

### 3.5 Применение

Кнопка **Apply** (или автоприменение при выходе с экрана):
- Перегенерация конфига с новыми значениями переменных и правил
- Сохранение в хранилище
- Если VPN запущен — snackbar «Restart VPN to apply changes»

## 4. Навигация

Оба экрана доступны из **drawer** на главном экране:

```
Drawer:
  ├── Subscriptions          — управление подписками
  ├── Settings               — переменные и правила
  ├── Config
  │   ├── Editor             — существующий ConfigScreen
  │   ├── Read from file     — (как есть)
  │   └── Paste from clipboard — (как есть)
  └── Debug                  — (как есть)
```

## 5. Цикл пользователя

1. Открыть Subscriptions → вставить URL → ноды загружены
2. (Опционально) Открыть Settings → настроить log level, DNS, clash API
3. Нажать «Generate config» или оно происходит автоматически
4. Вернуться на главный экран → Start VPN → ноды отображаются

## 6. Нецели

- Drag-and-drop для порядка подписок.
- Редактирование outbound selectors (wizard second tab).
- Редактирование DNS-серверов через drag-and-drop.
- Множественное редактирование подписок.

## 7. Критерии приёмки

- [ ] Можно добавить подписку по URL и увидеть количество нод.
- [ ] Можно добавить direct link и увидеть имя узла.
- [ ] Обновление всех подписок скачивает и парсит контент.
- [ ] Удаление подписки работает с перегенерацией конфига.
- [ ] Настройки (vars) отображаются по типам с корректными виджетами.
- [ ] Selectable rules переключаются и влияют на конфиг.
- [ ] Сгенерированный конфиг сохраняется и принимается ядром.
- [ ] Навигация из drawer работает.
