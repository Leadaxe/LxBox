## Платформа первой версии

**MVP v1 — только Android.**

## Допущения (чтобы не распухал scope)

| Допущение | Смысл |
|-----------|--------|
| **`experimental.clash_api` в конфиге** | Считаем, что в импортируемом JSON блок **есть и корректен**; отдельная проверка «включён ли Clash API» и отдельный UX «включите в конфиге» **не делаем**. |
| **Корректность JSON для sing-box** | Текст из буфера считаем **валидным** конфигом sing-box / libbox: достаточно **распарсить JSON** и передать ядру; отдельная схема-валидация и дружелюбные ошибки по полям **не входят** в MVP (при полном крахе парсинга — краткое сообщение «не удалось разобрать JSON»). |

## 1. Цель MVP

Минимальный сценарий на **одном главном экране**:

**Read** (вставить конфиг из буфера) → при необходимости **Start** (VPN) → выбрать **группу** → выбрать **узел** в списке и при необходимости **ping** одиночный → **Stop**.

Отдельные вкладки «Импорт», «Servers», «Диагностика» **не делаем** в MVP: всё перечисленное — на главном экране, кроме явно отложенного ниже.

### Договорённость по терминам (MVP)

В приложении **ровно один** сохранённый конфиг: **профиль** = **единственный конфиг** (заменяется при новом Read).

## 2. Главный экран (компоновка)

Сверху вниз:

1. **Кнопки:** **Start** / **Stop** (одна пара состояний VPN-туннеля) и **Read** — чтение текста из буфера обмена, сохранение как единственный конфиг (подмена предыдущего).
2. **Селектор групп** — выбор **selector-группы** (список из конфига, см. [`003`](../003%20servers%20tab/spec.md)).
3. **Список узлов** в выбранной группе: отображение имён, **текущий активный** outbound, действие **переключить на узел**, **одиночный ping** по узлу.

**В MVP нет:** массового ping, фильтров, сортировок, карты селекторов, экспорта URI, множественного выделения строк (детали — в `003`).

## 3. Что входит в MVP (по слоям)

### 3.1 Конфиг из буфера (Read)

- Чтение строки из clipboard по **Read**.
- Сохранение в локальное хранилище, замена предыдущего конфига.
- Парсинг JSON; при ошибке парсинга — короткое сообщение пользователю.

### 3.2 Туннель (Start / Stop)

- **Start** — поднять VPN с сохранённым конфигом (после выдачи системного разрешения VPN — см. §5).
- **Stop** — остановить туннель.
- Restart/reconnect — **не** обязательны в MVP.

### 3.3 Группы и узлы (Clash API)

- После запуска ядра — загрузка списка групп и узлов, переключение активного outbound в группе, одиночный ping — по [`003`](../003%20servers%20tab/spec.md).

### 3.4 Диагностика

- **Отдельный экран логов** (два потока: ядро / приложение) — **не** в MVP v1.
- Краткие **ошибки** подключения, запуска, switch, ping — **на главном экране** (строка состояния / snackbar), без сырого stack trace в UI (bridge — [`001`](../001%20mobile%20stack/spec.md)).

## 4. Навигация и первый запуск

- **Один** основной маршрут — главный экран; системный **Back** ведёт из любых системных диалогов на главный.
- **Первый запуск:** экран не перегружать; достаточно заголовка/краткой подписи («вставьте конфиг из буфера — Read, затем Start»). Расширенный онбординг — не в MVP.
- После **Read** при желании показывать краткое подтверждение «конфиг сохранён» (toast).

## 5. Разрешение VPN (Android)

- Разрешение **VPN** запрашивается в момент, когда пользователь нажимает **Start** (или по первой попытке старта), через системный диалог Android.
- При отказе — понятный текст, что без разрешения туннель недоступен; повторная попытка при следующем Start.
- Уведомление foreground service (если требуется для удержания VPN) — по правилам ОС и реализации native-слоя.

## 6. Явно не входит в MVP v1

| Область | Комментарий |
|---------|-------------|
| **iOS** | Позже. |
| Редактор JSON, импорт по URL/файлу | Нет. |
| Подписки по URL | Нет. |
| Вкладки / отдельный экран диагностики с логами | Нет (только краткие ошибки на главном). |
| Массовый ping, фильтры, сортировки, карта селекторов, экспорт | Нет (см. `003`). |

## 7. Критерии готовности MVP

- [ ] **Read** подставляет конфиг из буфера и сохраняет его как единственный; при битом JSON — короткая ошибка.
- [ ] **Start** / **Stop** управляют туннелем на Android; VPN permission обрабатывается предсказуемо.
- [ ] Селектор групп и список узлов работают с Clash API; переключение узла и **одиночный** ping — по [`003`](../003%20servers%20tab/spec.md).
- [ ] Ошибки видны пользователю в упрощённом виде на главном экране.
