# 021 — Built-in Speed Test

## Контекст

Пользователю нужно проверить скорость соединения через VPN-туннель, не выходя из приложения. Аналог — встроенный тест в Hiddify, Speedtest by Ookla.

Текущая реализация (MVP): один замер download/upload через Cloudflare/Hetzner, без выбора сервера, без истории.

## Концепция

Трёхфазный тест: **Ping → Download → Upload**, проходящий через активный VPN-туннель.

### Фазы теста

#### 1. Ping (латентность)
- HTTP GET на `gstatic.com/generate_204` (лёгкий, стабильный эндпоинт)
- 5 замеров, отбрасываем max и min, среднее из оставшихся 3
- Результат в мс

#### 2. Download
- Несколько параллельных потоков (4) для насыщения канала
- Прогрессивный размер: начинаем с малого файла (1MB) для калибровки, затем основной замер (10-25MB)
- Замер в реальном времени — обновление скорости каждые 500мс
- Таймаут: 15 секунд на фазу
- Fallback серверы: Cloudflare → Hetzner → custom

#### 3. Upload
- POST на Cloudflare `__up` endpoint
- Прогрессивный размер: 1MB калибровка, затем 5-10MB основной
- Аналогичный real-time замер

### Серверы для тестирования

| Приоритет | Провайдер | Download URL | Upload URL |
|-----------|-----------|-------------|------------|
| 1 | Cloudflare | `speed.cloudflare.com/__down?bytes=N` | `speed.cloudflare.com/__up` |
| 2 | Hetzner | `speed.hetzner.de/10MB.bin` | — |
| 3 | Custom | Настраивается пользователем | — |

- Анимированный прогресс-бар во время теста (real-time скорость)
- После завершения — итоговые цифры
- История последних 10 тестов (сохраняется в SharedPreferences)
- VPN-статус индикатор: показать через какой прокси идёт тест (или "Direct" если VPN выключен)

#### Анимация

- Цифры скорости обновляются в реальном времени (каждые 500мс)
- Прогресс-бар заполняется пропорционально прошедшему времени фазы
- Переход между фазами с плавной анимацией

### Хранение

В `SharedPreferences`:

```json
"speed_test_history": [
  {
    "timestamp": 1713200000,
    "ping": 42,
    "download": 85.3,
    "upload": 23.1,
    "proxy": "auto-proxy-out",
    "vpnEnabled": true
  }
]
```

Максимум 10 записей, FIFO.

### Точность замера

- **Multi-stream download**: 4 параллельных соединения для насыщения канала (аналог Speedtest)
- **Warm-up**: первый 1MB — калибровка, не учитывается в результате
- **Скользящее среднее**: результат = медиана по 500мс-интервалам, отбрасывая первый и последний
- **Ping**: 5 замеров, trimmed mean (без min/max)

## Улучшения vs текущая реализация

| Было (MVP) | Стало |
|------------|-------|
| 1 поток download | 4 параллельных потока |
| Один замер | Warm-up + основной замер |
| Статичный результат | Real-time обновление скорости |
| Нет истории | Последние 10 тестов |
| Нет контекста | Показывает через какой прокси тест |
| 10MB фиксированный | Адаптивный размер по скорости канала |

## Критерии приёмки

- [ ] Тест проходит через VPN-туннель когда VPN включен
- [ ] Download использует 4 параллельных потока
- [ ] Скорость обновляется в реальном времени (каждые 500мс)
- [ ] Ping — среднее из 5 замеров (trimmed mean)
- [ ] История последних 10 тестов сохраняется
- [ ] Отображается текущий прокси/direct при тесте
- [ ] Fallback на альтернативный сервер при ошибке
- [ ] Корректная работа при отсутствии интернета (ошибка, не крэш)
